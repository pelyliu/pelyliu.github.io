设计方法论 - 狩野分析（Kano Model）

本文最后更新于 2025-07-16T16:19:10+08:00

如何通过设计方法，来找出顾客对于产品各个功能的重要 / 满意程度

今天就要来介绍通过目前业界常用的顾客满意度模型中的 “狩野分析 / Kano Model” 来找出如何协助 “产品 / 功能 / 服务” 优化的方向。

你是否认为只要产品品质提升，顾客就会满意？在公司花了众多资源与时间改善客户提出的困扰后，客户就会给出了非常满意与正面的评价？有没有过发现优化需求后客户反而说这不是他们想要的？

狩野分析认为，要改善客户满意度 “多即是好” 这种不断增加新功能的取向，是一种无效的策略。但是设计团队可以通过调查与分析从中发掘哪些因素与特质是客户真正重视的，并决定孰轻孰重。将每个功能产品属性（例如：功能、价格、利益）归类的五个类别当中，便可以看出客户的满意度来自哪里。

在开始谈论狩野分析分析前。需要知道到底用户在想些什么。Nor Chen 老师提出在产品 / 服务的发展过程中，有一个固定的模式：

品质管制 → 品质管理 → 魅力创造

品质管制：你的产品功能或服务，必须达到基本要求。让人可以用。

品质管理：你的产品功能与服务，必须能够回应使用者的选择，并且维持一致的体验。

魅力创造：你的产品功能或服务，让人惊艳，但仅止于惊艳，而不会有强烈需求。

而这个模式其实就是使用 Kano Model 必须要具备的基本思维。在品质上要先达到基本要求（品质管制），接着进而满足客户需求（品质管理），而后才满足潛在需求（魅力创造）。

什么是 Kano Model

Kano Model 是狩野教授在 1984 年发表的满意度模型。用科学化的统计方法进行产品品质的主、客观分析。主要通过问卷获得数据，对数据分类得出相应结果。得出结果的企业，可以依照结果来区分用户对于产品功能的品质属性，并建立改善的优先顺序。

基本品质 → 一维品质 → 魅力品质 → 无差异品质

Kano Model 的主要用途

  • 辨认出有效提高客户满意度的功能或服务。

  • 分析产品功能、服务与客户满意度之间的相关性。

  • 依客户满意度研究，辨认产品功能的优先改善顺序。

Kano Model 的六大分类

情境：想像一下你下载了一个自拍的美颜 APP，然后选了一个模式自拍。

M: 必要（Must-be / 基本品质要素 / 基本型需求）：必要属性是产品必须具备的基本功能，一但确立后便不可或缺。属于这类别的功能，增加时可能不会增加客户的满意度，但如果缺少，一定会造成负面的影响。

发现自拍 APP 无法使用滤镜自拍：自拍 APP 要有美颜滤镜才是自拍 APP（不然好像用原生相机自拍就可以？）。这就代表，可以选择美颜滤镜是自拍 APP 的基本型需求。

O: 想要（One-Dimensional / 一维品质要素 / 期望型需求）：该属性与用户满意度呈线性关系。当具备这样的价值时用户会觉得产品更有价值；如果没有，价值则会降低。这类属性出现时，最好一直保持。

发现自拍 APP 可以美化各种部位：对于喜欢修图的用户（笔者）来说，可以针对照片修图就是期望型功能。当可以针对照片美化的功能越多。用户（笔者）就会越开心，但如果越少，用户（笔者）就会容易产生不满。

A: 惊艳（Attractive / 魅力品质要素 / 魅力型需求）：该属性类型的可以让用户感到愉悦会惊奇，有助于增进客户满意度。但是不具备此功能时通常也不会导致用户失望或沮丧。这类属性代表着潜在客户的需求。

发现自拍 APP 可以好友共编：虽然有没有好友共编不会影响到自拍 APP 的使用。但是对于用户（笔者）来说如果可以好友共编，那在跟好友自拍完后，就可以让好友自己调整，并且双方都可以下载照片。而不用在询问对方是否需要修图。因为此功能在没有时不会影响 APP 的使用，但有的话会带给用户惊喜。所以属于魅力型属性。

I: 中性（Indifferent / 无差异品质要素 / 无差异需求）：客户对于该类型的属性没有特别的感觉，不管有没有都不会对用户的满意度造成正面或负面的评价。

自拍 APP 的历史记录：对于用户（笔者）来说因为喜欢的相片都会下载至相册中。所以自拍 APP 的自拍历史记录对用户（笔者）来说，有或没有都没有差。故此功能为无差别属性。

R: 负面（Reserve / 反向品质要素 / 反向型需求）：产品应该避免的属性。此种品质充足时反而会引起顾客不满，不充足时才会满意。有些用户会愿意付出代价来避免接出该类型的属性的功能。或是选择没有该类型属性的产品。

发现自拍 APP 的照片会出现 Logo 浮水印：用户（笔者）不喜欢让别人知道自己使用了哪个美颜 APP，所以当 Logo 的浮水印出现在照片上时，对于用户（笔者）来说就是负面型属性。

Q: 无法辨认（Questionable / 疑问型需求）：此为有疑问之结果，一般情况下不会出现，除非问题问法不合理、受测者没理解问题、或在填写答案时写错等等。

Kano Model 的执行方法

狩野分析收集数据的方式是问卷调查，挑选出要进行比较与优化的服务后，分别调查用户对于提供（正向）与不提供（反向）某项服务的感受，每个问题有五个单选选项，范例如下图。

Kano Model 数据处理

整理回收的问券数据后，首先要将六大类中的第六类（无法辨认，Questionable / 疑问型需求）的类型样本筛掉后，才能进行数据转换。可以利用下方表格对照：

归类完所有答案的属性之后，统计单一问题中每个属性出现的次数，然后将每个属性的数值换算成百分比，来找出每个功能的属性。

例如：针对 XX 功能回复中 100 个样本内出现了 40 次”A 属性”，出现了 25 次”O 属性”则。XX 功能的代表属性为”A 属性”。（参考下表）

分析出每项功能的属性后，就可以用来做排序改善的优先顺序。根据上方提到的功能优先改善顺序

基本品质（M） → 一维品质（O） → 魅力品质（A） → 无差异品质（I）

改善功能的方向: 降低不满意度（移除 R 属性 & 改善 M 属性） > 提高满意度 （增加 O 属性 & A 属性）

在这里通常会搭配 Mike Timko 的 Better-Worse 敏感度公式来进行第二次分析。因为在有很多功能 / 服务要去做分析的情况，容易发现有很多功能的品质属性是一致的。这时候就可以通过 Better-Worse 敏感度公式分析图来决定优化处理顺序。

滿意度影响估算 (SI) = (A+O) / (A+O+M+I)

不满意度影响估算 (DSI) = (O+M) / (A+O+M+I) × -1

而正常来说 SI 会落在 0 到 1 之间，DSI 会落在 0 到 -1 之间，当 SI 和 DSI 的绝对值越大，代表其对改变服务的影响力越大。

沿用刚才的數據套入上面的公式：

SI = (40+25) / (40+25+15+8) = 0.738

DSI = (25+15) / (40+25+15+8) × -1 = -0.454

如果计算出来的结果让该功能落在于灰色范围内，则代表敏感性不大、可暂时不考虑的功能；反之离原点越远的优先程度越重。

Kano Model 的注意事项

（1）Kano Model 的答题过程中是由受测者来评估其对产品功能的喜好程度，因此要特别注意受测者对于功能的熟悉程度。如果是对 ToB 的客户，就不应该选择一般民众受测。而是要选择实际上会使用的目标群众进行测试。且如果该功能，在市面上并不常见，则受测者可能会因为没有相关的经验而难以作答。导致回收的问卷会容易出现偏差。因此在定义、设计问卷时要注意内容与受测者.

（2）使用者有可能随着时间与使用的产品经验上改变心态。所以如果产品的开发周期较长，可以定期针对产品功能进行检测。

（3）Kano Model 并无法看出使用者对某功能是否达到满意的程度。如果想要针对特定的功能找出使用者对于该功能的满意度。建议通过其他方式来找出答案.

（4）加入自评重要性量表（Self-stated Importance Scale）或者净推荐值（Net Promoter Score），可以获得更多使用者的想法，且功能的属性也许会有变化。所以在时间允许的情况下，发送问卷时也可以将自评重要性量表的题目放进去。
